# Novel Crawler & Online Reading System

This is a lightweight web novel crawler + local online reading system based on Flask.  
Simply provide the URL of a novel's **table of contents page** or **any chapter page**, and the system will automatically crawl the entire novel, save it as a `.txt` file, and generate an HTML version for reading.  
It includes a built-in library manager, supporting renaming, deletion, and online reading.

---

### ✨ Features

- 🕷️ **Intelligent Crawler**  
  - Automatically detects a table of contents page (extracts all chapter links) or a chapter page (follows "next chapter" links to crawl continuously)  
  - Supports paginated chapters (e.g., "Chapter 1 of 3") and merges content automatically  
  - Random User-Agent + retry mechanism + request delays to reduce blocking risk  
  - Real-time logging with persistent task status

- 📚 **Library Management**  
  - Lists all downloaded novels (shows filename, modification time, size, estimated chapter count)  
  - Supports **renaming** (custom display name) and **deletion**  
  - Metadata stored in `library_meta.json`; renaming does not affect the original file

- 📖 **Online Reader**  
  - Dynamically parses `.txt` files, splits chapters by `=====...` separators  
  - Table of contents navigation, previous/next chapter switching, responsive design  
  - Supports direct reading of HTML versions generated by the crawler (backward compatible)

- 🧵 **Multi‑Task Concurrency**  
  - Each crawl task runs in a separate background thread, not blocking the web service  
  - Task status and logs are updated in real time (polling API)

- ⚙️ **Production‑Ready**  
  - Provides Gunicorn and uWSGI configuration files for one‑click deployment  
  - Uses file locking (`fcntl`) to ensure task/metadata safety under multi‑process (Linux only)

---

### 🛠️ Tech Stack

- **Backend**: Python 3.12+, Flask, Requests, BeautifulSoup4  
- **Frontend**: HTML, CSS, JavaScript (vanilla, no frameworks)  
- **Deployment**: Gunicorn / uWSGI (optional)

### 📦 Installation Guide

1. **Clone the repository**

   ```bash
   git clone https://github.com/yourusername/novel-crawler-reader.git
   cd novel-crawler-reader
   ```

2. **Create a virtual environment (recommended)**

   ```bash
   python3 -m venv venv
   source venv/bin/activate   # Linux/Mac
   # or .\venv\Scripts\activate (Windows)
   ```

3. **Install dependencies**

   ```bash
   pip install -r requirements.txt
   ```

   > `requirements.txt` content:
   > ```
   > flask==2.3.3
   > werkzeug==2.3.8
   > requests>=2.25.0
   > beautifulsoup4>=4.9.0
   > ```

4. **Configuration (optional)**

   - **Output directory**: Default is `novels/` in the project root. You can change the `OUTPUT_DIR` variable in `webapp.py`.
   - **Port**: Default is `5000` in development. For production, modify `gunicorn_conf.py` or `uwsgi.ini`.

### 🚀 How to Run

#### Development / Debug Mode

```bash
python webapp.py
```
Visit `http://127.0.0.1:5000`

#### Production Deployment with Gunicorn

```bash
gunicorn -c gunicorn_conf.py webapp:app
```

#### Production Deployment with uWSGI

```bash
uwsgi --ini uwsgi.ini
```

> **Note**: In production, please run the service with a non‑root user (modify `uid` and `gid` in the config files) to avoid security risks.

### 📖 Usage

1. **Open the home page**: Visit `http://your-server:5000/`
2. **Submit a crawl task**:
   - Enter the starting URL (table of contents page or any chapter page)
   - Optionally specify a novel name (if left blank, it will be auto‑extracted from the `<title>` tag)
   - Click "Start Crawling"
3. **View task logs**: You will be redirected to the log page, showing real‑time progress. When the task finishes, the generated HTML file path will be displayed.
4. **Go to the library**: Click the "Library" link in the navigation bar to see all downloaded novels.
   - Click "Read" to read online (dynamic TXT parsing)
   - Click "Rename" to change the display name (does not affect the file name)
   - Click "Delete" to remove the novel and its metadata
5. **Online reading**:
   - On the reading page, a table of contents is shown on the left/top. Click a chapter to jump.
   - At the bottom of each chapter there are "Previous Chapter" / "Next Chapter" links.

### ⚠️ Important Notes

1. **Respect target websites**  
   Ensure your crawling activities comply with the target site's `robots.txt` and relevant laws. It is recommended to set reasonable delays to avoid putting pressure on the server.

2. **Anti‑scraping mechanisms**  
   Some websites may block frequent requests. This program already includes random User‑Agent, retry logic, and delays (2‑5 seconds). You may need to adjust `USER_AGENTS` and `base_delay` in `crawler.py` based on your target.

3. **Selector adaptation**  
   The default title/content/next‑page selectors work for many common novel sites. If a site has a special structure, you may need to modify the configuration variables in `crawler.py`, such as `COMMON_TITLE_SELECTORS`, `COMMON_CONTENT_SELECTORS`, etc.

4. **Concurrency and locking**  
   `tasks.json` and `library_meta.json` use `fcntl` file locks to ensure multi‑process safety. **This works only on Linux**. If you run on Windows, you will need to replace the locking mechanism (e.g., with `portalocker`) or avoid concurrent multi‑process writes.

5. **Encoding issues**  
   The crawler defaults to UTF‑8 decoding. If a website uses a different encoding, you may see garbled text. You can change `resp.encoding` in the `fetch_html` function of `crawler.py` or use `chardet` for automatic detection.

### ⚠️ 注意事项

1. **尊重目标网站**  
   请确保您的抓取行为符合目标网站的 `robots.txt` 及相关法律法规。建议设置合理延迟，避免对服务器造成压力。

2. **反爬机制**  
   部分网站可能对频繁访问进行封锁。本程序已内置随机 User-Agent、重试机制和延迟（2~5 秒），但仍需根据实际情况调整 `crawler.py` 中的 `USER_AGENTS` 和 `base_delay`。

3. **选择器适配**  
   默认的标题/内容/下一页选择器适用于常见小说网站。若目标网站结构特殊，可能需要修改 `crawler.py` 中的 `COMMON_TITLE_SELECTORS`、`COMMON_CONTENT_SELECTORS` 等配置。

4. **并发与锁**  
   `tasks.json` 和 `library_meta.json` 使用 `fcntl` 文件锁保证多进程安全，**仅支持 Linux**。若在 Windows 下运行，需替换为其他锁机制（如 `portalocker`）或避免多进程并发写入。

5. **编码问题**  
   爬虫默认以 UTF-8 解码，若网站编码不同可能乱码。可修改 `crawler.py` 中 `fetch_html` 里的 `resp.encoding` 或使用 `chardet` 自动检测。
